# Input

## Overview

Inputs allow users to enter text into a UI. They typically appear in forms and dialogs.

## Usage

- It is always recommended to use a placeholder in an input field.
- Use the `password` type for password inputs to hide the value..
- The assistive text should be provided when:
  - The field is required
  - The error occurs

### Password

The `type="password"` hides the entered value. Clicking on the eye icon will show the value.

<Canvas>
  <Story name="Password">
    <Input label="Label" type="password" placeholder="Enter password" />
  </Story>
</Canvas>

### With hooks

This example uses the `useState` hook to store the value entered by the user.

<Canvas>
  <Story name="With hooks">

      return (
        <>
          <Input
            onChange={({ target }) => setValue(target.value)}
            value={value}
            label="Label"
            placeholder="Enter text"
          />
          <div>Entered value: {value}</div>
        </>
      );
    }}
